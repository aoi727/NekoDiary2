# NekoDiary 仕様書

## 1. 概要
NekoDiary は、PySide6 で構築されたデスクトップ向け日記アプリケーションです。  
SQLite データベースを利用して日記データを保存し、日記の作成、編集、削除、検索、閲覧、設定変更、バックアップ保存を行えます。

## 2. 動作環境
- 実装言語: Python
- UI フレームワーク: PySide6
- データベース: SQLite
- 設定保存: QSettings

## 3. 起動仕様
### 3.1 起動時の処理
- アプリ起動時に `style.qss` を読み込みます。
- 前回使用したデータベースパスを設定ファイルから取得します。
- 保存済みパスが存在する場合はそのまま使用します。
- 保存済みパスが存在しない場合は、以下を選択するダイアログを表示します。
  - 既存の DB ファイルを選択
  - 新しい DB ファイルを作成
  - キャンセル

### 3.2 対応する DB ファイル拡張子
- `.db`
- `.sqlite`
- `.sqlite3`

## 4. データ仕様
### 4.1 テーブル構成
アプリは SQLite 内に以下のテーブルを持ちます。

#### Category
- `id`: INTEGER, 主キー, 自動採番
- `name`: TEXT, 必須, 一意

#### Memo
- `id`: INTEGER, 主キー, 自動採番
- `date`: TEXT, 必須
- `category`: TEXT, 必須, 既定値 `''`
- `title`: TEXT, 必須
- `content`: TEXT, 必須, 既定値 `''`

#### AppConfig
- `key`: TEXT, 主キー
- `value`: TEXT, 必須

### 4.2 初期データ
- Category テーブルが空の場合、以下のカテゴリを自動登録します。
  - `日常`
  - `仕事`
  - `天気`
  - `読書`
  - `旅行`
  - `雑記`
- Memo テーブルが空の場合、サンプル日記 2 件を登録します。
- AppConfig テーブルが空の場合、以下の初期値を登録します。
  - `app_title = My Diary`
  - `font_size_mode = 中`
  - `home_background_path = ""`
  - `home_background_opacity = 30`
  - `editor_background_path = ""`
  - `editor_background_opacity = 30`

### 4.3 データモデル
#### Memo
- `id`: int
- `date`: str
- `category`: str
- `title`: str
- `content`: str

#### Category
- `id`: int
- `name`: str

#### AppConfig
- `app_title`: str
- `font_size_mode`: str
- `home_background_path`: str
- `home_background_opacity`: int
- `editor_background_path`: str
- `editor_background_opacity`: int

### 5.2 メニューバー

### 5.3 設定画面

- 文字の大きさ
- ホーム画面背景画像
- ホーム画面背景濃度
- 編集画面背景画像
- 編集画面背景濃度

### 5.4 日記編集画面
日記編集画面では以下を編集できます。

- 日付
- タイトル
- 本文
- カテゴリ
- 文字サイズ
- 文字色
- 太字装飾
- 画像挿入

## 6. メイン画面機能
### 6.1 ツールバー
ツールバーには以下の機能があります。

- `ホーム`
  - 現在日付を選択
  - 期間を `全期間` に設定
  - キーワード検索をクリア
  - カテゴリ選択を初期化
- `新規作成`
  - 新しい日記作成ダイアログを開く
- `表示・編集`
  - 選択中の日記を編集する
- `削除`
  - 選択中の日記を削除する
- `日記ファイルの保存`
  - 現在の DB ファイルと画像フォルダを ZIP 形式でバックアップ保存する
- `設定`
  - 設定画面を開く

### 6.2 検索条件
#### 期間検索
以下の期間で日記一覧を絞り込みできます。

- `全期間`
- `1か月前まで`
- `3か月前まで`
- `6か月前まで`
- `1年前まで`

#### カテゴリ検索
- Category テーブルに登録されているカテゴリから選択できます。
- 未選択時は `<選択>` を表示します。

#### キーワード検索
- タイトルまたは本文を対象に部分一致検索を行います。

### 6.3 補助ボタン
左カラムには以下の補助ボタンがあります。

- `新しいものを表示`
- `条件をクリア`
- `検索`

### 6.4 日記一覧
- 条件に一致する日記を日付降順、同日内は ID 降順で表示します。
- 各一覧項目には以下を表示します。
  - 日付
  - タイトル
  - 本文プレビュー
- 本文プレビューはプレーンテキスト化し、56 文字超過時は `...` を付与します。

### 6.5 日記プレビュー
- 選択中の日記のタイトル、日付、カテゴリ、本文を表示します。
- 本文は HTML を整形して表示します。

## 7. 日記編集機能
### 7.1 新規作成
- 新規作成ダイアログを開きます。
- 保存時は Memo テーブルに新規追加します。
- カテゴリ入力内容から新規カテゴリがあれば Category テーブルへ追加します。

### 7.2 編集
- 選択中の日記の内容を読み込んで編集ダイアログを開きます。
- 保存時は既存レコードを更新します。

### 7.3 削除
- 選択中の日記を削除できます。
- 削除前に確認ダイアログを表示します。

### 7.4 編集項目
- 日付
- タイトル
- 本文
- カテゴリ

### 7.5 本文編集
- リッチテキスト入力に対応します。
- 本文には HTML を保存します。

### 7.6 文字装飾
- 文字サイズを選択可能です。
  - `10`
  - `12`
  - `14`
  - `16`
  - `18`
  - `20`
  - `24`
  - `28`
  - `32`
- 文字色を選択可能です。
- 太字を切り替え可能です。

### 7.7 画像挿入
- 画像ファイルを選択して本文へ挿入できます。
- 対応拡張子は以下です。
  - `.png`
  - `.jpg`
  - `.jpeg`
  - `.bmp`
- 選択画像は DB ファイルと同じフォルダにある `img` フォルダへコピーします。
- 保存ファイル名は `yyyyMMdd_ランダム文字列.拡張子` 形式です。
- 本文には画像の URI を挿入します。
- 保存時には表示用 URI を相対保存用パスへ置換します。

### 7.8 カテゴリ入力仕様
- カテゴリは `;` または空白で区切って複数指定できます。
- `#` プレフィックスがあっても保存時に正規化されます。
- 全角空白は半角空白へ変換して扱います。
- 保存時は重複を除去し、`; ` 区切りで保存します。

## 8. 設定機能
### 8.1 設定項目
- 日記タイトル
- 文字の大きさ
- ホーム画面背景画像
- ホーム画面背景濃度
- 編集画面背景画像
- 編集画面背景濃度

### 8.2 文字サイズ設定
全体文字サイズは以下の 4 段階です。

- `小 = 9pt`
- `中 = 10pt`
- `大 = 12pt`
- `特大 = 14pt`

旧設定値との互換として、以下は自動変換されます。

- `小さめ -> 小`
- `標準 -> 中`
- `大きめ -> 大`

### 8.3 文字サイズ適用範囲
- アプリケーション全体
- メイン画面
- 設定画面
- 日記編集画面
- メイン画面配下の子ウィジェット一式

### 8.4 背景画像設定
- ホーム画面背景
- 編集画面背景

それぞれ以下の設定を持ちます。

- 画像ファイルパス
- 不透明度

### 8.5 背景濃度
- 0 から 100 の範囲で設定可能です。
- 5 刻みで変更可能です。

### 8.6 設定の保存先
設定内容は AppConfig テーブルに保存します。

## 9. 設定ファイル仕様
QSettings により以下を保存します。

- `main/geometry`
- `editor/geometry`
- `settings/geometry`
- `database/path`

設定ファイルは `settings.ini` として起動時に読めるアプリの場所に保存されます。  
開発時はプロジェクト直下、Windows 実行ファイルでは実行ファイルと同じフォルダを使用します。

## 10. バックアップ保存機能
### 10.1 機能概要
`日記ファイルの保存` は、現在使用中の DB ファイルと DB ファイル横の `img` フォルダを ZIP 形式で保存するバックアップ機能です。

### 10.2 保存処理
保存時の処理順序は以下です。

1. 現在の DB 接続をクローズ
2. DB ファイルと `img` フォルダ内の画像を ZIP ファイルへ格納
3. DB を再接続

### 10.3 保存先指定
- 保存ダイアログを表示します。
- 既定ファイル名は `元ファイル名_backup_yyyyMMdd_HHmmss.zip` です。
- ZIP 内には DB ファイルをルートに、画像を `img/` 配下に格納します。

### 10.4 制約
- 元の DB ファイルと同じパスは保存先に指定できません。
- 保存失敗時はエラーダイアログを表示します。
- 再接続失敗時もエラー内容を表示します。

## 11. アイコン・画像リソース仕様
### 11.1 読み込み方式
- アプリ内のアイコン画像は `Sozai` フォルダから直接読み込みます。
- `resources.qrc` は使用しません。

### 11.2 主な使用箇所
- アプリウィンドウアイコン
- ツールバー
- 設定画面ボタン
- 編集画面ボタン
- 検索補助ボタン

## 12. DB 接続仕様
- DB 接続は `sqlite3.connect()` で作成します。
- `row_factory` は `sqlite3.Row` を使用します。
- アプリ終了時に DB 接続をクローズします。
- バックアップ保存時に一時的にクローズ、保存後に再接続します。

## 13. ステータス表示
ステータスバーには以下のようなメッセージを表示します。

- `準備完了`
- `データファイル : <DBパス>`
- `日記を保存しました (ID n)`
- `日記を更新しました (ID n)`
- `日記を削除しました (ID n)`
- `日記バックアップを保存しました: <保存先>`
- `日記バックアップの保存に失敗しました`

## 14. エラー時の挙動
- 編集対象未選択時は案内ダイアログを表示します。
- 削除前は確認ダイアログを表示します。
- バックアップ保存失敗時はエラーダイアログを表示します。
- DB ファイル未選択または無効時は再選択を促します。

## 15. 現在の制約
- 本文保存形式は HTML ベースです。
- 画像は外部ファイルとして DB ファイルと同じフォルダの `img` に保存します。
- テスト仕様や配布仕様は本書では未定義です。
